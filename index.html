1. Overview

This Privacy Policy describes how Scientific Lightning Solutions handles information used with its RAG Tool Custom GPT.

The RAG Tool is a local Retrieval-Augmented Generation system for building searchable knowledge bases from project documentation, source code, generated autodocs, product manuals, schemas, and other technical files.

The purpose of the RAG Tool is to turn scattered project knowledge into a searchable, source-backed assistant that helps users ask practical questions about a codebase or documentation set.

2. Purpose of the Tool

The RAG Tool is designed to help users understand complex projects faster by retrieving and synthesizing relevant context from approved project materials.

The tool may help answer questions such as:

  • What is this project for?
  • Where should a new developer start?
  • Which files are involved in a workflow?
  • Where is a feature implemented?
  • What might break if a file or schema changes?
  • How do user-facing docs relate to raw implementation details?
  • Which manuals, specs, or source files support an answer?

The goal of the tool is not to create perfect fully automated documentation. The goal is to create usable, searchable, source-backed knowledge that helps engineers, developers, technicians, and report writers understand complex projects faster.

3. Information the Tool May Process

The RAG Tool may process information that users or project maintainers provide for use in a RAG project or knowledge base. Based on the tool description, this may include:

  • Project documentation
  • Source code
  • Generated autodocs
  • Product manuals
  • Schemas
  • Technical files
  • User questions submitted to the Custom GPT
  • Retrieved context used to answer user questions
  • Generated answers, summaries, explanations, and documentation drafts

4. How Information Is Used

Scientific Lightning Solutions uses information processed by the RAG Tool to support retrieval, search, summarization, explanation, synthesis, navigation, and documentation drafting.

The RAG Tool uses Python code for deterministic parsing, graph building, file ingestion, indexing, and retrieval. AI models are used for summarization, explanation, synthesis, navigation, and documentation drafting. Human users provide high-level project intent, verify important outputs, and maintain critical context.

5. Custom GPT Action Support

The RAG Tool includes Custom GPT Action support for external tool use. When the Custom GPT uses the RAG Tool, user questions or related request information may be sent to the RAG Tool so it can search the relevant project knowledge base and return source-backed context.

6. Retrieval Modes

The RAG Tool supports multiple retrieval modes, including docs, code, autodoc, mixed, and auto. These modes are used to retrieve relevant context from the available project knowledge base.

7. Local FastAPI Server

The RAG Tool includes a local FastAPI server with endpoints for one-shot RAG answers and retrieval-only agentic RAG. These endpoints are used to support the tool's search and question answering workflows.

8. Knowledge Base Materials

Materials added to a RAG project may be ingested, indexed, searched, summarized, and used as source-backed context for answers. The README describes the tool as being able to work with project documentation, source code, generated autodocs, product manuals, schemas, and other technical files.

9. AI-Assisted Outputs

The RAG Tool may produce summaries, explanations, synthesized answers, navigation guidance, and documentation drafts based on retrieved context.

The README states that the tool is not intended to replace human review. Users should verify important outputs before relying on them.

10. Model Use

The README describes model fallback across Gemini/Gemma and OpenRouter models. Information included in prompts, retrieved context, or generated summaries may be processed by the configured models as part of the tool's summarization, explanation, synthesis, navigation, and documentation drafting workflows.
